Implementação: Definir uma Métrica de Desempenho¶

difcil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso geralmente feito utilizando algum tipo de mtrica de desempenho, atravs do clculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida til. Para este projeto, voc ir calcular o coeficiente de determinao, R2, para quantificar o desempenho do seu modelo. O coeficiente de determinao uma estatstica til no campo de anlise de regresso uma vez que descreve o quo "bom" a capacidade do modelo em fazer estimativas.

Os valores para R2 tm um alcance de 0 a 1, que captura a porcentagem da correlao ao quadrado entre a estimativa e o valor atual da varivel alvo. Um modelo R2 de valor 0 sempre falha ao estimar a varivel alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a varivel alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da varivel alvo (ao utilizar o modelo) que pode ser explicada pelos atributos. Um modelo pode dar tambm um R2 negativo, que indica que o modelo no melhor do que aquele que estima ingenuamente a mdia da varivel alvo.

Questão 3 - Treinamento e Teste¶

  • Qual o benefcio de separar o conjunto de dados em alguma relao de subconjuntos de treinamento e de teste para um algoritmo de aprendizagem?

Dica: O que pode dar errado se no houver uma maneira de testar seu modelo?

Resposta:

Segundo o Guia do Usurio Sklearn: "Aprender os parmetros de uma funo de previso e test-la nos mesmos dados um erro metodolgico: um modelo que apenas repetiria os rtulos das amostras que acabaram de ver teriam uma pontuao perfeita, mas no conseguiriam prever nada de til no entanto."

Dessa forma, o benefcio de separar o conjunto de dados em subconjuntos de treinamento e teste a possibilidade de validao, alm de estimar a acurcia do mesmo. No realizar essa diviso resulta em modelos de Alta Varincia (overfitting), que como aponta a citao acima, so modelos que se ajustam muito bem ao conjunto de dados anteriormente observado, mas que se mostram ineficaz para prever novos resultados.

Em resumo, considera-se essencial realizar a diviso do conjunto dados, especialmente aplicando a variao cruzada k-fold, que permite reciclar nossos dados, alm de impedir modelos de Alto Vis (underfitting). Variao cruzada abordada na sesso "Avaliando o Desempenho do Modelo", neste notebook.

Resposta:

A Validao Cruzada K-fold um mtodo que permite, atravs da reciclagem, a utilizao de todos os dados disponveis para treinamento e teste. Este mtodo maximiza os pontos de treinamento, permitindo que modelo compreenda melhor os dados, resultando em uma generalizao mais eficiente e conseguentemente em uma melhor acurcia.

A tcnica consiste em separar os dados em k pedaos de tamanhos iguais e realizar k experimentos de treinamento e teste. Cada pedao utilizado uma vez como validao, enquanto os k-1 pedaos restantes formam o conjunto de treinamento. Por fim calculado a mdia do resultado.

aconselhvel utilizar o embaralhamento dos k pedaos, evitando assim vis no modelo. Em sklearn, a funo KFold de model_selection permite tal ao atravs do parmetro shuffle=True.

from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.model_selection import GridSearchCV

def fit_model(X, y):
    """ Desempenhar busca em matriz sobre o parmetro the 'max_depth' para uma 
        rvore de deciso de regresso treinada nos dados de entrada [X, y]. """
    
    # Gerar conjuntos de validao-cruzada para o treinamento de dados
    cv_sets = ShuffleSplit(n_splits=10, test_size = 0.20, random_state=40)

    # TODO: Gerar uma rvore de deciso de regresso de objeto
    regressor = DecisionTreeRegressor()

    # TODO: Gerar um dicionrio para o parmetro 'max_depth' com um alcance de 1 a 10
    params = {'max_depth':range(1, 11)}
    
    # TODO: Transformar 'performance_metric' em uma funo de pontuao utilizando 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Gerar o objeto de busca em matriz
    grid = GridSearchCV(estimator = regressor, param_grid = params, scoring = scoring_fnc, cv = cv_sets)

    # Ajustar o objeto de busca em matriz com os dados para calcular o modelo timo
    grid = grid.fit(X, y)

    # Devolver o modelo timo depois de realizar o ajuste dos dados
    return grid.best_estimator_
